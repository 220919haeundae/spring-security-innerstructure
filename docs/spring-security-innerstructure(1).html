스프링 시큐리티(Spring Security) 구조

시큐리티 의존성이 없는 경우

클라이언트 → WAS(톰캣)[필터] → Spring Container → controller

시큐리티 의존성 추가된 경우
  1. WAS의 필터에 하나의 필터를 만들어서 넣고 해당 필터에서 요청을 가로챔
  1. 스프링 컨테이너 내부에 구현되어 있는 스프링 시큐리티 감시 로직을 거침
  1. 시큐리티 로직을 마친 후 다시 WAS의 다음 필터로 복귀

시큐리티 로직 구성

여러개의 필터들이 나열된 필터 체인 형태로 구성되어 있음

각각의 필터에서 csrf, 로그아웃, 인가 등 여러 작업을 수행한다.

  • 시큐리티 필터 체인 : 일련의 과정들을 수행하는 필터들의 묶음, 여러개의 필터 체인을 가질 수 있음

주요 용어
  • DelegatingFilterProxy : 스프링 Bean을 찾아 요청을 넘겨주는 서블릿 필터
  • FilterChainProxy: 스프링 시큐리티 의존성을 추가하면 DelegatingFilterProxy에 의해 호출되는 SecurityFilterChain들을 가지고 있는 Bean
  • SecurityFilterChain: 스프링 시큐리티 필터들의 묶음으로 실제 시큐리티 로직이 처리되는 부분, FilterChainProxy가 SecurityFilterChain들을 가지고 있

FilterChainProxy(springSecurityFilterChain)

DelegatingFilterProxy에서 전달받은 요청을 내부에 있는 SecurityFilterChain들을 거쳐서 인증/인가를 거쳐서 내부에 있는 controller로 전달함

SecurityFilterChain 등록
  • 커스텀 SecurityFilterChain 등록
    • 스프링 시큐리티 의존성을 추가하면 기본적인 DefaultSecurityFilterChain 하나가 등록됨
@Configuration
@EnableWebSecurity
public class SecurityConfig {

	@Bean
	public SecurityFilterChain filterChain1(HttpSecurity httpSecurity) throws Exception {
		return httpSecurity.build();
	}
	
	@Bean
	public SecurityFilterChain filterChain2(HttpSecurity httpSecurity) throws Exception {
		return httpSecurity.build();
	}
	
	...
	
}
  • SecurityFilterChain은 FilterChainProxy에 등록되어 내부적인 인증/인가 절차를 수행하는데, 이 때 요청은 모든 SecurityFilterChain을 거치는 것이 아니라 securityMatchers 메서드에 등록되어 있는 경로에 대해서 인증/인가 등의 절차를 거치게 되어 있다. 여러 필터에 동일한 경로가 등록되어있더라도, FilterChainProxy 내부에 등록된 순서대로 가장 먼저 해당 경로를 매핑하는 SecurityFilterChain 하나를 거치게 된다.

@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    @Order(1)
    public SecurityFilterChain securityFilterChain1(HttpSecurity httpSecurity) throws Exception {

        httpSecurity
                .securityMatchers(auth -> auth
                        .requestMatchers("/user"));

        httpSecurity
                .authorizeHttpRequests(auth -> auth
                        .requestMatchers("/user").permitAll());
        return httpSecurity.build();
    }

    @Bean
    @Order(2)
    public SecurityFilterChain securityFilterChain2(HttpSecurity httpSecurity) throws Exception {

        httpSecurity
                .securityMatchers(auth -> auth
                        .requestMatchers("/admin"));

        httpSecurity
                .authorizeHttpRequests(auth -> auth
                        .requestMatchers("/admin").permitAll());
        return httpSecurity.build();
    }

    @Bean
    public WebSecurityCustomizer webSecurityCustomizer() {
        return web -> web.ignoring().requestMatchers("/img/**");
    }

}
  • 이 때 @Order 어노테이션은 SecurityFilterChain이 FIlterChainProxy에 등록되는 순서를 정한다.
  • 맨 아래 WebSecurityCustomizer는 인증/인가 등 security의 내부적 인증 작업에서 제외할 요청 경로를 지정할 수 있다.
